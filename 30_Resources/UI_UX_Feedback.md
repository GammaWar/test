# UI & UX Feedback (Sammlung) & Kurzbeschreibung
Sammelbecken für Verbesserungsvorschläge zur Benutzeroberfläche und Benutzererfahrung.

## Basisdaten

| Bereich | Vorschlag / Problem | Priorität |
| :--- | :--- | :--- |
| **Kampf** | Einheiten-Auswahl (Massenauswahl optimieren) | Mittel |
| **Bauwesen** | Anzeige der Gesamtzeit in der Bauliste | Niedrig |
| **Benachrichtigung** | Externe Benachrichtigungen (E-Mail/Push) bei Angriffen | Hoch |
| **Organisation** | Bessere Sortierung von Blueprints in Ordnern | Mittel |
| **Navigation** | Kategorisierung von Missionen (Tabellenansicht) | Mittel |

## Mechanik / Verhalten

### Detaillierte Vorschläge

#### 1. Kampf-UI
Es wurde vorgeschlagen, eine Funktion zur Massenauswahl oder zum schnellen Setzen aller Einheiten auf das Maximum zu implementieren, um den Klickaufwand in Kämpfen zu reduzieren.

#### 2. Benachrichtigungssystem (Alerts)
Spieler wünschen sich Benachrichtigungen außerhalb des Spiels (E-Mail, Push, Telegram/WhatsApp), insbesondere bei bevorstehenden Angriffen. 
* **Hinweis:** Es gibt Diskussionen über das Balancing (Vermeidung von "Ninja-Bots" und Erhalt der Notwendigkeit zum "Saven").

#### 3. Bauwesen & Logistik
Eine Anzeige der kumulierten Gesamtzeit aller aktuell in der Warteschlange befindlichen Konstruktionen am Ende der Liste würde die Planung erleichtern.

#### 4. Organisation (Blueprints)
Die aktuelle Ordnerstruktur für Blueprints wird als unübersichtlich empfunden (z. B. Diskrepanz zwischen Streitkräften und Schiffswerft). Eine bessere Kategorisierung/Sortierung ist gewünscht.

#### 5. Missions-Übersicht
Um die Übersichtlichkeit bei vielen Missionen zu erhöhen, wird eine tabellarische Darstellung nach Kategorien (z. B. "Abholen", "Transport") vorgeschlagen.

## Tests & Hypothesen
* **Benachrichtigungen:** Ein zu aggressives Benachrichtigungssystem könnte das Risiko-Management des Spiels (das "Saven") entwerten.
* **UI-Komplexität:** Mehr Informationen (wie Gesamtbauzeit) müssen so integriert werden, dass die UI nicht überladen wirkt.
